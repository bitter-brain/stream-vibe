# StreamVibe

**StreamVibe** is a front-end project that recreates the interface of a modern streaming service.  
The main goal of the project is to practice building complex UI components and a scalable architecture using **vanilla JavaScript** and modern development tools.

The application includes multiple interactive interface elements such as tabs, sliders, a responsive mobile menu, a custom video player, input masks, and a custom select component. All functionality is implemented with **pure JavaScript**, without using front-end frameworks.

## Features

- Responsive **mobile navigation menu** with burger toggle
- Interactive **tabs**
- Multiple **sliders and carousels**
- **Custom video player**
- **Custom Select component**
- **Input masks** for form fields
- Accessibility improvements using **WAI-ARIA**
- Fully responsive layout based on a **Figma design**

## Tech Stack

- **JavaScript (ES6+)**
- **JSX templates** (without React)
- **Vite + Minista** for static site generation
- **Sass (SCSS)** for styling
- **PostCSS** with automatic `px → rem` conversion
- **BEM** methodology for CSS class naming

## Architecture & Development

The project follows a **component-based approach** implemented with **Object-Oriented Programming in JavaScript**.  
Each UI element is designed as a reusable module to keep the codebase clean and maintainable.

During development, several modern browser APIs were used:

- **Proxy**
- **ResizeObserver**
- **MatchMedia**

The project also focuses on **semantic HTML, accessibility, and performance**, while closely following the design layout from Figma.

## Goal of the Project

The purpose of this project is to gain hands-on experience with:

- building complex UI components from scratch
- structuring large front-end projects without frameworks
- working with modern build tools
- improving accessibility and responsive design
